# Day65 Heap - UAF 2: Leak Primitive

## Goal

dangling pointer를 이용해 새로 재할당된 객체의 pointer field를 읽고 heap 주소를 leak하는 흐름을 확인했다.

## Flow

```c
Obj *a = malloc(sizeof(Obj));
Obj *alias = a;

free(a);
a = NULL;

Obj *b = malloc(sizeof(Obj));
b->ptr = secret;

printf("%p\n", alias->ptr);

a를 free해도 alias에는 old heap 주소가 남아 있다.
이후 같은 size class로 b가 할당되면 같은 chunk가 재사용되어 alias == b가 된다.

따라서:

alias->ptr == b->ptr
b->ptr == secret

이므로 alias->ptr을 출력하면 secret의 heap 주소가 leak된다.

Result
alias == b
alias->ptr == b->ptr
alias->ptr leak → secret heap address
Primitive

이번 실습의 primitive는 UAF 기반 heap leak primitive다.

Failure Case

b를 다른 size class로 할당하면 같은 chunk가 재사용되지 않을 수 있다.

Obj *b = malloc(0x100);

이 경우 alias != b가 되어 alias->ptr로 b->ptr을 leak하는 재현은 실패할 수 있다.

Summary
UAF write는 새 객체의 값을 덮는 primitive다.
UAF read는 새 객체 내부 값을 읽는 primitive다.
이번 실습에서는 dangling alias로 b->ptr을 읽어 heap 주소를 leak했다.
heap leak은 safe-linking 우회나 heap exploit 준비 단계에서 중요하다.
