##day43
pie nx on 상태에서 ret2win

##목표
pie base를 구하고 bof를 이용해 win 함수 호출

프로그램 1차 입력에서 %1$p를 입력 주소값형태 출력 fsb 확인 - pie base leak 가능
2차 입력에서 a200개 입력 Program received signal SIGSEGV, Segmentation fault 오류 출력 bof확인

find 프로그램으로 pie base 후보를 추린다
후보 선정기준
- 0x5로 시작하는 값
- objdump로 확인한 main win vuln 의 offset을 뺐을때 마지막자리가 000으로 정렬되는가
pie idx 후보 4개로 추림

pie idx 확정
gdb에서 vulv에 브레이크 vmmap으로 r-xp 범위 확인 실행가능한 코드영역 범위
후보 인덱스 입력 출력 확인
확정 기준
- 출력된 값이 r-xp 범위 안에 있는가
- main win vuln offset을 뺐을 때 r-xp바로 위에있는값 즉 pie base와 같은가
이번 실습에서는 main offset을 뺐더니 000으로 정렬 즉 main leak 이다
pie idx 확정

##payload 구조
1차 - pie leak, pie idx를 넣어 값을 leak 
leak된 값에 main offset을 뺌 pie base

2차 - gdb에서 패턴을 넣어 알아낸 offset = 136, disas vuln에서 확인한 rbp - 0x80 버퍼시작부터 saved rbp 까지의 거리는 0x80인데
카나리가 없기 때문에 rbp 8바이트를 더하면 10진수로 136 패턴을 넣어서 알아낸 값과 동일 offset 확정
패딩 + win

##실패 케이스
첫번째 시도에서는 ret을 추가하지 않고 시도했다
결과는 win 함수 진입성공 WIN 출력 하지만 열린 쉘에서 id 명령 입력 실패

##원인
rsp 정렬을 확인 win 함수 진입시에 rsp % 16의 값이 0x0 인것을 확인 정상적인 함수 진입시 rsp%16은 0x8 이어야 한다

##해결
rsp 정렬을 맞추기 위해 ret가젯 추가
패딩 + ret + win
